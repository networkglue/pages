Network Glue

Work is underway to complete a working prototype of

Network Glue

.

Network Glue

is, or is planned to be, a centralization and consolidation application for different network platforms. When completed,

Network Glue

will manage user accounts, user groups, network devices, end points and device configurations from the different platforms through a single user interface.

Philosophy

Obviously, we're not Glue Networks. The name is purely a coincidence. We believe existing management platforms should not be replaced but harnessed. Our goal is to make sure different tools work together and are streamlined and synchronized. p class="text-info" style="display:inline">Network Glue

is not a management platform in itself but more a manager of managers. Our features are only limited by the APIs provided by the other platforms. We plan to modularize the different platform components for easy integration and configuration.

Supported Platforms

Current platforms targetted are Cisco ACS 5.x, Cisco ISE 2.x, Help Systems Intermapper 5.x & 6.x, HP Network Automation 9.x & 10.x. Future platforms may include integration with LDAP, Microsoft Active Directory & IBM Tivoli.

Technology

Network Glue

is built with Perl, Moose, Mojolicious, DBIx::Class, Postgresql, Bootstrap3 & jQuery.

Integration is based on the following Perl modules: Net::Cisco::ACS (REST), Net::Cisco::ISE (REST), Net::Intermapper (HTTP), Net::HP::NA (SOAP).

Release Information

will be released as free software under the GPLv3 license, with commercial support and integration available at a later stage.
